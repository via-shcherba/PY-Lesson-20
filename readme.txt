### Описание функций

1. **/start**: Приветствует пользователя и предлагает использовать /help.
2. **/help**: Показывает список доступных команд.
3. **/fetch**: Запускает парсер в отдельном потоке, чтобы не блокировать основной поток, и уведомляет пользователя о завершении.
4. **/view**: Позволяет просматривать кэшированные вакансии, если они есть.

### Обработка долгих запросов

Чтобы избежать блокировки, парсинг осуществляется в отдельном потоке с помощью `threading.Thread`. 
Пользователь получает уведомление, если парсер уже работает. Кэш вакансий позволяет пользователю просматривать результаты, 
пока не будет запущен новый парсинг.
